## Context

First minor-feature release after v1.0.2. Bundles Phase 1 of the
venue-and-merch-links initiative — official venue page link on every
ShowDetail. Per the staged-release plan adopted 2026-05-06, v1.0.3
ships small (one feature) so any regression is unambiguous.

Held back intentionally:
- Email MFA → v1.0.4 (security-sensitive, deserves its own release
  signal for support/diagnostics)
- Buddies Phase 2, recommendations, Wrapped map, contacts, etc. →
  v1.1+

## What's in v1.0.3

**Venue links** — every show's ShowDetail now has a pill labeled
with the venue name that links to the official venue website
(resolved via Wikipedia/Wikidata, with a Google search fallback for
venues without Wikipedia coverage). Per
`2026-05-05-venue-and-merch-links.md`.

**Email confirmation on signup** — new accounts must enter a
6-digit code sent to their email before the account is active.
Closes the signup-with-typo-locks-out-real-owner hole, prerequisite
for upcoming contacts/discoverability features. Per
`2026-05-06-email-mfa.md` Phase 1.

Files added:
- `src/web/components/OtpEntry.jsx` — 6-box OTP entry with paste-
  to-fill, auto-advance, 60s resend cooldown, iOS auto-fill support
  via `autocomplete="one-time-code"`
- `supabase/migrations/0006_venue_url.sql` — `venue_url` column

Files changed:
- `src/web/lib/auth.js` — `verifyEmailOtp`, `resendSignupOtp`
- `src/web/pages/auth/SignUp.jsx` — routes to OtpEntry on
  unconfirmed signup
- `src/web/pages/auth/SignIn.jsx` — detects `email_not_confirmed`
  error, auto-resends, routes to OtpEntry
- `src/web/lib/db/shows.js` — `venueUrl` mapping; `venue_url`
  omitted from row when empty
- `src/web/api.js` — `lookupVenueUrl` (Wikipedia/Wikidata),
  `venueSearchUrl` fallback
- `src/web/components/ShowDetail.jsx` — auto-resolver, single-shot
  effect, stable updateShow ref
- `src/web/pages/LogShow.jsx` — `venueUrl` state + clear-on-edit
- `src/web/App.css` — `.detail-link-pill` family, `.otp-row`/`.otp-box`

  unblocked. Venue-links code reworked twice during dev test:
  - `372d4db` — switched from Ticketmaster venue URLs (which point
    to Ticketmaster's own listing pages, not official sites) to
    Wikipedia/Wikidata resolution for the actual official venue

    (prefix-only) → `query&list=search` (full-text), added
    Wikipedia external-links fallback for venues with empty

    deep links like `/event/...` resolve to the venue homepage.
  - `401130b` — broke the infinite loop in ShowDetail's auto-
    resolver. Effect now runs once per show.id change with a
    stable ref for `updateShow`, and `db/shows.js` only sends
    `venue_url` in the row when populated (so missing-migration
    state degrades gracefully).
- 2026-05-06 (later still): user ran the migration in Supabase

## Open questions / follow-ups

- **Marketing**: this is the first feature release worth a real
  push. Mojo Reel possibility — "Tap any venue" 6-second demo. Track
  in marketing/Reels follow-ups, not here.
- **Watch the Ticketmaster venue match rate** in production once
  shows start saving with a `venue_url`. If <50% of new logs land a
  URL, the strict-match logic in `lookupVenueUrl` is too tight and
  needs loosening (or a Setlist.fm fallback).
- **Empty-state UX**: the "Venue page not found" pill is a dead-end.
  Future polish: surface a Google search link as a last resort.
