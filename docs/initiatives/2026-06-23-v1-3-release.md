# v1.3 Release — "the app is ready to market"

- Started: 2026-06-23
- Status: in-progress (ready to build)
- Last updated: 2026-06-23

## Context
1.2.2 (build 20) is **live**. Everything below is committed to `main` but not yet in
a shipped build, so it all rides in the next build = **1.3**. We deliberately kept 1.3
**lean** so it can ship this week and unblock harder marketing — at ~50 users the
bottleneck is distribution, not features. Depth/growth features that don't gate
marketing (comparative-rating, the going social loops, the native "stickier" bundle)
were pushed to **1.4**. See the roadmap in this folder's `README.md`.

## Plan
### In 1.3 (all already built + committed)
- **Share-card redesign** — 5 styles (Vibe/Poster/Marquee/Player/Ticket) + the
  Share-view builder + real PNG export. [[Share Card Redesign]]
- **Get Started onboarding** — first-run 3-step activation checklist on Home
  (log → add friends → alerts), replacing the old zero-show block + taste prompt.
  [[cold-start-activation]]
- **First-show reward** — the share card auto-opens after the user's first log.
  [[cold-start-activation]]
- **Premium polish pass** — warm-only fallback gradient app-wide, poster letter-art +
  shadow, richer home hero, sign-in/onboarding ember halo, entrance motion.
  [[Premium Polish Pass]]
- Plus the social/utility work already on `main` from prior sessions (friends feed,
  likes/comments, discover by artist/genre, music taste, log-any-band) — to the extent
  it wasn't already bundled into 1.2.2.

### Ship checklist
1. Bump version 1.2.2 → 1.3 in Xcode (`MARKETING_VERSION`; build 20 → 21).
2. `npm run build && npx cap sync ios`.
3. **Confirm the backend is deployed** so taste/social features actually work:
   - migrations 0011 (social) / 0012 (friendship guard) / 0013 (profile taste) applied
   - `tour-alerts` edge function (+ cron) and `notify-*` functions deployed
4. Xcode → Archive → App Store Connect → submit; write a "What's New" leading with the
   new-user experience + share cards.

### Explicitly NOT in 1.3 (→ 1.4)
- Comparative rating (Beli-style) — [[Comparative Rating System (Beli-style)]]
- Invite-on-going + friends-going push — [[going-social-loops]]
- The native "stickier & more accessible" bundle: home-screen widget,
  [[live-activities-show-day]], [[social-auth]], dark mode. See README roadmap.
